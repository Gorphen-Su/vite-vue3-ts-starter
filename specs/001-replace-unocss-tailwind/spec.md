# 功能规范：UnoCSS 迁移至 Tailwind CSS

**功能分支**: `001-replace-unocss-tailwind`
**创建时间**: 2026-03-06
**状态**: 草稿
**输入**: 用户描述：优化脚手架，使用 tailwind 工具替换当前 unocss 工具，并同步修改当前环境配置

## 用户场景与测试

### 用户场景 1 - 开发环境样式开发体验 (优先级：P1)

开发者在使用脚手架进行开发时，需要继续使用原子化 CSS 编写样式，包括类名写法、@apply 指令、主题定制等。

**优先级原因**: 这是核心开发体验，直接影响日常开发效率。

**独立测试**: 可以在开发模式下启动项目，验证现有样式类名是否仍然生效。

**验收场景**:

1. **给定** 一个使用现有 UnoCSS 类名的 Vue 组件，**当** 启动开发服务器时，**那么** 所有样式应该正确渲染
2. **给定** 一个使用 `@apply` 指令的样式块，**当** 编译时，**那么** 应该正确生成 CSS

---

### 用户场景 2 - 主题配置迁移 (优先级：P1)

项目现有的自定义主题配置 (颜色、断点、自定义规则) 需要在新的工具中保持一致。

**优先级原因**: 主题配置是项目视觉一致性的基础，必须保持兼容。

**独立测试**: 检查现有组件使用的主题颜色和断点是否正确应用。

**验收场景**:

1. **给定** 使用 `text-primary`、`bg-success` 等主题色的组件，**当** 渲染时，**那么** 颜色值应与原有定义一致
2. **给定** 响应式布局使用 `sm:` 断点的组件，**当** 在不同屏幕尺寸下，**那么** 断点行为应与原来一致

---

### 用户场景 3 - 构建产物样式完整性 (优先级：P1)

生产环境构建后，所有样式应该正确打包和压缩。

**优先级原因**: 直接影响线上产品的视觉效果。

**独立测试**: 执行生产构建并部署预览，验证所有页面样式正常。

**验收场景**:

1. **给定** 执行 `pnpm build` 命令，**当** 构建完成时，**那么** 不应该有样式相关的错误或警告
2. **给定** 构建后的应用，**当** 在浏览器中运行时，**那么** 所有组件样式应该与开发环境一致

---

### 边界情况

- 当自定义规则 (如 `navbar-shadow`) 在迁移后如何保持兼容？
- 当图标预设 (presetIcons) 在 Tailwind CSS 中如何处理？
- 当属性化选择器模式 (presetAttributify) 的写法是否需要调整？

## 需求

### 功能需求

- **FR-001**: 系统必须使用 Tailwind CSS 替代 UnoCSS 作为原子化 CSS 引擎
- **FR-002**: 系统必须保留现有的主题颜色定义 (primary、success、warning、danger、info)
- **FR-003**: 系统必须保留现有的断点配置 (sm: 600px)
- **FR-004**: 系统必须支持 `@apply` 指令在 Vue 组件中使用
- **FR-005**: 系统必须保留自定义规则 `navbar-shadow` 的功能
- **FR-006**: 系统必须保留背景图片快捷规则 `bgimage-{dir}-{name}.{ext}` 的功能
- **FR-007**: 系统必须移除 UnoCSS 相关的依赖和配置
- **FR-008**: 系统必须配置 Tailwind CSS 的 Vite 插件
- **FR-009**: 系统必须更新构建配置以支持 Tailwind CSS 的 CSS 注入

### 关键实体

- **主题配置**: 包含颜色、断点、自定义样式的定义集合
- **转换器**: 处理 `@apply` 指令的编译工具
- **内容检测**: 扫描源代码中使用的类名以生成最终的 CSS 文件

## 成功标准

### 可衡量的结果

- **SC-001**: 所有现有组件的样式在迁移后 100% 保持视觉一致
- **SC-002**: 开发服务器启动时间不超过原有时间的 150%
- **SC-003**: 生产构建产物中 CSS 文件大小不超过原有的 120%
- **SC-004**: 构建过程不产生任何 CSS 相关的错误或警告
- **SC-005**: 开发者无需修改现有组件中的类名即可正常工作

## 依赖关系

- **DEP-001**: 需要安装 Tailwind CSS 及其 Vite 插件
- **DEP-002**: 需要安装 `@tailwindcss/vite` 或等价的官方插件
- **DEP-003**: 可能需要额外的插件来支持 UnoCSS 特有的功能 (如图标预设)

## 假设

- **ASS-001**: Tailwind CSS v3.x 或 v4.x 版本将用于替换
- **ASS-002**: 假设项目中的样式写法大部分符合 Tailwind CSS 的命名约定
- **ASS-003**: 假设 UnoCSS 的 presetWind3 预设与 Tailwind CSS 的工具类高度兼容
- **ASS-004**: 图标功能将由其他方式 (如 unplugin-icons) 继续支持，因为 Tailwind CSS 本身不提供图标预设

## 范围

### 包含

- UnoCSS 配置迁移到 Tailwind CSS 配置
- Vite 插件配置更新
- 主题配置转换
- 自定义规则转换
- 开发环境和生产环境验证

### 不包含

- 不修改现有组件的样式类名写法 (除非不兼容)
- 不改变 Element Plus 的集成方式
- 不调整现有的 ESLint 和 Stylelint 规则 (除非必要)
